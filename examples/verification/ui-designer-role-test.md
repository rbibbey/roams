# UI Designer Role Test

Use this exercise to validate UI Designer behavior in isolation.

## Goal

Create design guidance for a simple landing page without implementing code.

Project brief:

`Design a single-page landing experience for ROAMS that feels practical, modern, and builder-oriented. It should work on desktop and mobile, avoid generic SaaS styling, and communicate calm operational competence.`

## What The UI Designer Should Do

- interpret the project theme and tone
- identify platform and screen-size constraints
- define visual and interaction principles
- provide implementation-ready guidance
- identify any assets or tools that would materially help

## Expected Output

The UI Designer should return:

- theme summary
- design principles
- layout direction
- typography and color guidance
- responsive considerations
- implementation notes or asset recommendations

## Pass Criteria

- design starts from theme and constraints
- guidance is specific enough for implementation
- desktop and mobile considerations are addressed
- the output avoids generic styling advice
- open design questions are surfaced clearly

## Common Failure Signals

- jumping straight to components without a theme
- generic or interchangeable UI direction
- no responsiveness considerations
- output too vague for a coder to use

# Candidate Memory

Purpose: stage potentially reusable information before it is promoted into long-term memory.

Use this file when information seems useful but is not yet clearly durable enough for `style.md`, `decisions.md`, or `lessons.md`.

Rules:

- Keep entries short and evidence-based.
- Include the intended target bucket when known.
- Promote, defer, or discard candidates during review.
- Remove candidates after they are promoted or discarded.
- Do not use this file for task-local notes that belong in `.agent/PLANS.md`.

## Review Workflow

For each candidate, decide one of:

- `promote`: move it into the correct long-term memory bucket
- `defer`: keep it here until more evidence appears
- `discard`: remove it because it is too weak, stale, or redundant

## Candidate Template

### Candidate Title

- state: proposed | deferred
- target bucket: style | decisions | lessons | unknown
- source: task, verification run, postmortem, or repeated observation
- evidence: short factual reason this might matter again
- next review trigger: repetition, next similar task, or scheduled review

# Decision Memory

Purpose: record important project-level decisions and why they were made.

Promotion rules:

- Record decisions that affect future implementation.
- Include enough rationale to avoid repeating the same debate.
- Skip temporary or obvious choices.

## Decisions

### v1 operating model

- Decision: start with a simple Manager-led hierarchy and lightweight memory system.
- Why: a small, durable operating model is easier to reuse and refine than a complex framework.

### roles before subagents

- Decision: define specialist roles as operating modes first and add real subagents later.
- Why: this keeps the system simple while preserving a path to parallel execution.

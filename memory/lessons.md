# Lessons Memory

Purpose: store reusable lessons from repeated successes or failures.

Promotion rules:

- Prefer lessons that prevent future mistakes or improve repeatable workflows.
- Promote immediately when the lesson is clearly durable and high value.
- Otherwise wait for repetition before adding it.

Do not log routine activity here.

## Lessons

- Start simple and evolve the operating model from real usage rather than imagined complexity.
- Begin UI work with theme and constraints before generating components or assets.
- Keep memory selective; excess noise makes future decisions worse, not better.

# Style Memory

Purpose: capture stable preferences about how work should be approached.

Promotion rules:

- Add only durable preferences.
- Prefer short statements over explanation.
- Remove or revise entries that no longer reflect actual practice.

## Current Preferences

- Prefer minimal, reviewable changes over broad rewrites.
- Prefer implementation-first iteration: build, verify, refine.
- Favor clarity, maintainability, and simplicity before optimization.
- Keep explanations concise and practical unless deeper detail is requested.
- Use specialist roles intentionally; do not add process overhead without value.

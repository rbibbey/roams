# Skill Evolution Ops

Use repeated evidence to improve the operating model without waiting for manual prompting.

## Workflow

1. Review the latest retrospective, memory-review output, and repeated user or reviewer feedback.
2. Identify whether the same omission, confusion, or rework pattern has happened more than once.
3. Decide the smallest durable remedy.
4. Route the remedy to the correct target.
5. Return a clear evolution decision for the Manager.

## Remedy Targets

- no change
- stage candidate improvement
- patch shared skill guidance
- patch project-local skill guidance
- add or refine verification coverage
- propose roadmap update

## Decision Rules

- Prefer tightening an existing skill over adding a new role.
- Use verification updates when the miss should have been caught rather than prevented only by better prompting.
- Use roadmap updates when the capability boundary itself has changed.
- Do not treat one-off frustration as durable process evidence unless it reflects a larger pattern.

## Required Outputs

- repeated pattern summary
- evidence threshold judgment
- selected remedy target
- brief rationale

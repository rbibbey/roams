# Purpose

This repository provides a reusable ROAMS starter template for running software projects through role-based orchestration, selective memory, explicit documentation and retrospective gates, skill-driven workflows, and lightweight automation guidance.

# Current Stage

The project is in an early post-v1 stage with the core operating model, role skills, research capability, roadmap governance, subagent orchestration, bootstrap automation, and explicit commit-readiness gate capabilities now documented.

# Near-Term Goals

The next steps are to validate the new documentation, memory, retrospective, and skill-evolution loops against real work, refine cross-repo evaluation prompts, and keep automation bounded until repeated usage proves broader autonomy is safe.

# Open Questions

The main open questions are how much of the evaluation loop should be encoded as reusable automation artifacts first, how aggressively the skill-evolution loop should patch shared skills automatically, and which cross-repo issue-planning patterns become durable enough to standardize.
